## 2024-05-24 - [Interactive Elements and Screen Reader Status Updates]
**Learning:** Found a pattern where dynamic text elements updating without user focus (like quantum state status messages) are completely ignored by screen readers, and interactive scrollable regions (like event histories) are unreachable via keyboard. Additionally, the project's secondary text default color (`#999` on `#f5f5f5` or `#fff`) fails WCAG AA contrast ratios, and disabled buttons retain `pointer` cursors with full visual weight, causing confusion.
**Action:** Always wrap dynamically updating text with `role="status"` and `aria-live="polite"`. Add `tabIndex={0}` and `role="log"` to scrollable data lists to enable keyboard access. Use at least `#555` for secondary text against light backgrounds, and strictly apply `opacity: 0.5` with `cursor: "not-allowed"` on disabled buttons.
