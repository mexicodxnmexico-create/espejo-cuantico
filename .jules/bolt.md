## 2024-05-18 - React Three Fiber Parent State Re-render
**Learning:** In React Three Fiber (R3F), storing rapidly changing state (e.g., animation intensity via `setInterval` and `useState` tracking every 1-2 seconds) in a parent component causes the entire `Canvas` and all its 3D children to re-render constantly. This is a massive performance bottleneck.
**Action:** Always internalize rapidly changing values into child components using `useRef` and R3F's `useFrame` hook to update Three.js object properties directly (e.g., `material.emissiveIntensity` or `scale`), completely bypassing React's render cycle for animations.
